# Paint-a-Booth

Photo booth web application. SvelteKit frontend deployed to Cloudflare Workers with SQLite (libSQL) database and Better Auth for authentication. The `firmware/` directory is reserved for future hardware/firmware components.

## Technology Stack

- **Framework:** SvelteKit 2 (Svelte 5) with TypeScript
- **Styling:** Tailwind CSS v4 (Vite plugin)
- **Auth:** Better Auth (email/password, Drizzle adapter, SvelteKit cookies plugin)
- **Database:** SQLite via libSQL (`@libsql/client`) + Drizzle ORM
- **Hosting:** Cloudflare Workers (`@sveltejs/adapter-cloudflare`)
- **Package Manager:** pnpm
- **Linting:** ESLint 9 (flat config) + Prettier
- **Build:** Vite 7

### Key Files

| File | Purpose |
|------|---------|
| `website/src/hooks.server.ts` | Attaches `session` and `user` to `event.locals` via Better Auth |

| `website/src/lib/server/db/schema.ts` | Drizzle schema — app tables + re-exported auth tables |

## Essential Commands

All commands run from the `website/` directory.

```bash
cd website

# Install dependencies
pnpm install

# Development server
pnpm dev

# Type checking
pnpm check              # one-shot
pnpm check:watch        # continuous

# Lint & format
pnpm lint               # prettier --check + eslint
pnpm format             # prettier --write

# Build & preview
pnpm build              # vite build
pnpm preview            # build + wrangler dev (local Cloudflare preview)

# Deploy
pnpm deploy             # build + wrangler deploy

# Database
pnpm db:generate        # generate Drizzle migration files
pnpm db:push            # push schema directly (dev)
pnpm db:migrate          # run migrations
pnpm db:studio          # open Drizzle Studio

# Auth schema (re-generate after changing Better Auth config)
pnpm auth:schema

# Cloudflare bindings typegen
pnpm cf-typegen
```

## Patterns

### Authentication

Session data is attached in `hooks.server.ts` and available via `event.locals.session` / `event.locals.user` in server load functions and actions.

### Database Schema Changes

1. Edit `website/src/lib/server/db/schema.ts`.
2. Run `pnpm db:generate` to create a migration, or `pnpm db:push` for quick dev iteration.
3. If Better Auth config changed, run `pnpm auth:schema` to regenerate `auth.schema.ts`.

### Server-Only Code

Place server-only modules under `src/lib/server/`. SvelteKit prevents client-side import of `$lib/server/*`.

## Anti-Patterns

- **Do not** edit `auth.schema.ts` manually — it is auto-generated by `pnpm auth:schema`.
- **Do not** edit `worker-configuration.d.ts` manually — run `pnpm cf-typegen`.
- **Do not** import from `$lib/server/` in client-side code.

## Code Style

- **Tabs** for indentation (not spaces).
- **Single quotes**, no trailing commas, 100-char print width (see `.prettierrc`).
- Prettier plugins: `prettier-plugin-svelte`, `prettier-plugin-tailwindcss`.
- ESLint: flat config with `typescript-eslint` recommended + `eslint-plugin-svelte`.
- `no-undef` rule is disabled (TypeScript handles this).
